# FEF v1.1.0 Release Notes

Release date: 2026-07-09

Tag: `v1.1.0`

## Summary

FEF v1.1.0 is the first evaluation-backed release of the Framework for Engineering Excellence.

This release turns FEF from a prompt/documentation framework into a measured enterprise-output quality framework. It keeps the frozen architecture intact while adding runtime routing, model-role guidance, proposal consistency review, and golden-test evaluation assets.

FEF still does not claim to increase model intelligence. Its purpose is to improve repeatable engineering behavior:

- better context framing
- stronger evidence discipline
- calibrated claims
- more consistent proposal quality
- better technical consistency review
- lower output variance

## Architecture Status

The frozen architecture is preserved:

```text
Core Kernel
→ Policies
→ Modules
→ Domains
→ Reviewers
→ Workflows
→ Golden Tests
→ User Request
```

No new permanent layer was added.

## Major Changes

### Runtime Entry

`CLAUDE.md` now acts as a runtime entry point rather than a duplicate rule container.

It points to:

- `kernel/CoreKernel.md`
- `kernel/MetaRules.md`
- `kernel/Checklist.md`
- `docs/context-protocol.md`
- `docs/model-usage.md`
- `docs/loading-map.md`

### Loading Map

Added `docs/loading-map.md` with:

- task-to-pack routing
- load limits
- simple low-risk task bypass
- one-pass reviewer rule
- missing-pack fail-safe behavior

### Context and Model Usage

Added:

- `docs/context-protocol.md`
- `docs/model-usage.md`

These define how to convert surface requests into operational problems and how to separate Builder, Reviewer, and Architect model roles.

### RHEL Domain Pack Expansion

`domains/RHEL.md` now includes guidance for:

- version-sensitive claims
- lifecycle and subscription scope
- public-sector considerations
- disconnected/restricted network operation
- security and compliance claims
- operations and automation
- competitive/clone OS positioning
- proposal source requirements

### Proposal Consistency Review

Added proposal consistency review behavior across:

- `modules/Proposal.md`
- `reviewers/ProposalReviewer.md`
- `workflows/ProposalWorkflow.md`
- `docs/loading-map.md`

The review mode checks:

- internal contradictions
- requirement coverage gaps
- terminology/version/date/number inconsistencies
- claim/evidence alignment
- hidden dependencies and exclusions
- roadmap/scope mismatch
- severity classification
- targeted fixes without unnecessary rewrite

### Golden Tests

Operationalized:

- `tests/GoldenTest-001.md` — RHEL public-sector proposal generation
- `tests/GoldenTest-011.md` — proposal consistency checking

Added:

- fixed prompts
- fixtures
- answer key for GT011
- runner protocols
- result worksheets
- raw output storage

## Evaluation Results

### GT001: RHEL Proposal Generation

| Variant | Average | Std Dev | Range | Result |
|---|---:|---:|---|---|
| Baseline | 45.8 | 29.0 | 14-83 | |
| FEF | 84.6 | 0.5 | 84-85 | PASS |

Observed improvement:

- Average uplift: +38.8 points
- Variance sharply reduced
- FEF consistently identified ambiguity and contradiction
- FEF used `[unverified]` discipline more reliably

Main remaining weakness:

- Factual accuracy remained capped because external source verification was not performed during the test.

### GT011: Proposal Consistency Check

| Variant | Average | Std Dev | Range | Result |
|---|---:|---:|---|---|
| Baseline | 54.8 | 11.6 | 38-72 | |
| FEF | 85.6 | 3.2 | 82-91 | PASS |

Observed improvement:

- Average uplift: +30.8 points
- Variance significantly reduced
- FEF improved requirement coverage and severity classification
- FEF consistently surfaced domain/compliance validation needs

Main remaining weakness:

- Some no-downtime and hidden dependency issues were still missed or under-framed.

## Follow-Up Fixes Included

After evaluation, v1.1.0 includes targeted fixes for the observed gaps:

- stronger evidence-sensitive claim handling
- stricter RHEL proposal source requirements
- hidden dependency and appendix-only exclusion surfacing
- roadmap/scope/dependency mismatch review guidance

## What v1.1.0 Means

This release is an initial evaluated release.

It demonstrates that FEF can improve output consistency and review quality in the tested proposal scenarios.

It does not prove universal quality improvement across all domains or task types.

## Known Limits

- Evaluation sample size is small: 5 baseline and 5 FEF runs per golden test.
- Results are strongest for proposal generation and proposal consistency review.
- Source-backed factual accuracy still requires external verification workflow.
- GT012/GT013 are not yet implemented for operations manuals and RCA.
- Public GitHub release metadata may need to be created separately from this document.

## Recommended v1.2 Direction

Prioritize measured expansion, not prompt growth.

Recommended next golden tests:

1. GT012 — RHEL operations manual / SOP quality
2. GT013 — Linux/RHEL RCA quality

Recommended practical work:

- apply FEF to a real customer proposal or internal proposal draft
- evaluate factual source verification workflow
- test Opus/Sonnet role separation on real enterprise artifacts
