# Proposal Workflow

1. Frame customer problem.
2. Identify audience.
3. Select domain pack.
4. Build proposal outline.
5. Draft business value.
6. Draft technical value.
7. Add risks and mitigations.
8. Add roadmap.
9. Review with Proposal Reviewer.
10. Finalize.

## Consistency Check Workflow

Use when the user asks to review or check an existing proposal.

1. Identify proposal purpose, audience, and evaluation context.
2. Identify stated requirements, assumptions, and constraints.
3. Map requirements to proposal sections.
4. Check internal consistency across summary, body, tables, figures, appendix, and conclusion.
5. Check terminology, product names, versions, dates, quantities, and scope statements.
6. Check claim/evidence alignment.
7. Flag technical, security, compliance, or operational claims that need domain validation.
8. Classify issues by severity.
9. Recommend targeted fixes.
10. Stop after one review pass unless the user requests rewrite or re-review.
