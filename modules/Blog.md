# Blog Module

## Purpose

Create technical blog posts with practical depth and low AI-like tone.

## Structure

1. Hook
2. Problem
3. Background
4. Technical Explanation
5. Practical Example
6. Operational Notes
7. Conclusion

## Rules

- Avoid generic introductions.
- Use concrete examples.
- Keep claims grounded.
- Prefer practitioner voice.
## S-Core OSSLab Style Mode

Use this mode when writing practical Korean technical blog posts inspired by S-Core OSSLab Insight/Trends style.

Observed style characteristics:

- Use a practical, approachable Korean title that makes the value obvious.
- Prefer question, number, comparison, or result-driven titles when natural.
- Connect trends to real operational concerns, not abstract hype.
- Explain enough background for readers who know the domain but may not know the specific tool or issue.
- Move from problem -> concept -> example -> operational implication.
- Use concrete terms, commands, metrics, diagrams, or scenarios where helpful.
- Avoid vendor brochure tone and unsupported superlatives.
- Keep the tone professional but readable; avoid stiff translated prose.

Title patterns:

- "왜 ...인가?"
- "... 한눈에 보기"
- "... 완전 정복"
- "... 작동 원리"
- "... 실제 적용/구축/테스트"
- "... 정말 가치가 있을까요?"
- "... 했는데, 진짜 ... 될까?"
- "숫자로 보는 ..."

Recommended structure:

1. Hook: why this topic matters now
2. Problem: what engineers or organizations struggle with
3. Concept: core technical idea in plain language
4. Deep dive: architecture, flow, mechanism, or trade-off
5. Practical example: command, configuration, scenario, benchmark, or checklist
6. Operational notes: validation, monitoring, security, cost, rollback, or limitations
7. Common pitfalls or misconceptions
8. Conclusion: what to adopt, verify, or watch next

Rules:

- State product/version scope when relevant.
- Separate confirmed facts from interpretation.
- Mark unsupported numeric, lifecycle, benchmark, certification, or cost claims as `[unverified]`.
- If using a performance number, include test condition or source requirement.
- If the post is trend-focused, explain the practical impact on enterprise adoption or operations.
- If the post is how-to-focused, include validation and failure symptoms where useful.
- Do not copy wording from source articles; use the observed structure and editorial pattern only.
